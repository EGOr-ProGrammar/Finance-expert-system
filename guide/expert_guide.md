# Руководство эксперта (Knowledge Engineer Guide)

Данный документ описывает процесс настройки и управления базой знаний финансовой экспертной системы.

## 1. Введение
Экспертный модуль (`FuzzyExpertSystem_Expert`) — это инструмент для формализации человеческих знаний в виде нечетких множеств и логических правил. Система использует модель Мамдани-Заде.

## 2. Работа с переменными

### 2.1 Входные переменные (Фаззификация)
Для каждой входной переменной необходимо определить лингвистические термы и функции принадлежности (MF).

**Доступные типы функций:**
1.  **Гауссовская (Gaussian):**
    *   `c` (Center): Точка, где степень принадлежности равна 1.0.
    *   `sigma` (Sigma): Ширина "колокола" (крутизна).
2.  **Трапецеидальная (Trapezoidal):**
    *   `c` (Center): Геометрический центр фигуры.
    *   `t` (Top Width): Ширина верхнего основания (где MF = 1.0).
    *   `s` (Slope): Крутизна боковых сторон.

### 2.2 Выходные переменные (Дефаззификация)
Для выходных переменных (результатов) система использует метод **среднего центра**.
*   Вам не нужно задавать сложные кривые.
*   Для каждого терма (например, "Рекомендовать покупку") укажите одно числовое значение — **центр**. Это "идеальное" значение данного решения на числовой оси.

## 3. Формирование правил
Правила объединяют входные данные в логический вывод.

**Структура:** `ЕСЛИ (Переменная1 есть ТермА) И (Переменная2 есть ТермБ) ТО (Выход есть ТермГ)`.

**Рекомендации:**
*   Убедитесь, что имена переменных и термов в правилах в точности совпадают с теми, что вы создали ранее.
*   Старайтесь покрыть правилами весь диапазон возможных входных значений, чтобы избежать ситуации, когда "ни одно правило не сработало".

## 4. Сохранение и файлы
*   Все данные хранятся в `data/knowledge_base.json`.
*   При добавлении данных они сохраняются в оперативной памяти. Чтобы записать их на диск, выберите пункт **"Save and Exit"**.
*   Вы можете вручную редактировать JSON-файл, если знакомы с его структурой, но использование экспертного модуля гарантирует валидность данных.
